## 一、软件是用来做什么的

「域名小能手」帮你**把"想造一个站"这件事从设计到上线全部走完**，整个流程不需要再切换任何其他工具：

| 你想做什么 | 软件能帮你做到 |
| :--- | :--- |
| 拿一个新域名，从零设计一个落地页 | AI 自动生成 HTML/CSS/Logo，过程中可继续对话改样式 |
| 一次性给几十个域名套同样的模板，做矩阵站 | 输入域名列表 → 自动复制 + 替换品牌字 + 重新生成 Logo |
| 需要走公司流程才能买域名 / 加解析 | 在对话里说一句话，自动嵌入一张飞书审批卡片 |
| 站做好了想直接上线 | 选目标服务器，点「部署」，自动上传 + 配 Nginx + reload |
| 想看历史造过的站、复盘部署日志 | 顶部三个 Tab 切换查看、预览、批量管理 |

> 提示：所有产出的网站文件都在你电脑上，离开这台机器看不到；走的飞书审批和上线动作会实时反馈到这个应用里。

## 二、安装与启动

1. 双击 IT 同学给你的 **安装包**（Windows 是 `.exe`，Mac 是 `.dmg`）。
2. 按系统提示完成安装；安装结束后桌面会出现 **「域名小能手」** 图标。
3. 双击启动 → 进入登录页（详见下一节）。

> 如果你拿到的是绿色版（免安装版），直接双击文件夹里的可执行文件即可。

> 💡 **配置文件**：第一次启动后，软件会自动在安装目录生成一份 `desktop.config.json`。这里面装的是 **API 密钥 / 飞书企业凭据 / 服务器登录信息**，IT 同学会预先帮你填好；如果需要换服务器或更新凭据，找 IT 同学或参考运维手册。

## 三、第一次打开 · 登录飞书

**为什么要登录飞书？**
所有审批、站点上线后通知、"我的审批"都要绑定到你的飞书账号，才能精确地把状态推送到你这边。**没有登录的话整个应用都用不了。**

操作步骤：

1. 打开软件，看到登录卡片，点击紫色的 **「登录飞书」** 按钮。
2. 系统会自动唤起浏览器跳到飞书登录页。
3. 在浏览器里完成飞书账号登录 + 同意授权。
4. 浏览器会显示 **"登录成功，可以关闭此页面回到桌面应用"**，回到应用即可。
5. 之后再次启动应用通常无需重复登录；若长时间未使用、token 过期，会重新弹出登录页。

## 五、场景一：让 AI 给我做一个全新的网站

### 适用什么时候

- 拿到一个**新域名**，想让 AI 从零做一个落地页；
- 不想自己设计，也不要参考模板，全部交给 AI；
- 做完后还想**继续追加修改要求**（比如「Hero 的标题改成…」）。

### 操作步骤

1. 顶部点 **「建站」** Tab。
2. 在底部输入框正下方，把 **模式胶囊** 切到 **「AI 全量」**。
3. 在输入框里**第一行写域名**，**从第二行起描述你想要的网站长什么样**，例如：
   > myshop.com
   >
   > Build a high-converting landing page for a SaaS shop platform. Hero: bold headline + screenshot. Sections: features (3 cards), pricing (3 tiers), FAQ (5 items), final CTA. Style: minimal, dark mode.
4. 按回车（或点右侧"发送"）。
5. **左侧的对话窗里会开始实时输出 AI 的思考与生成过程**，过程中你可以：
   - 在顶部点 **「停止」** 中途打断；
   - 等它停止后追加一条新消息，AI 会**在原网站基础上做增量修改**；
   - 任何时候去 **「网站库」** 找到这个域名，点预览即可看到当前效果。

预留截图位：

<p align="center">
  <img src="./images/screenshot-buildchat-ai.png" alt="AI 全量模式：输入框 + 流式输出对话气泡" width="800" />
</p>

### 你不需要关心的事

- 行业风格：默认就是「程序化广告 / 数字营销」风格；
- 语言：网站文案默认全部英文；
- Logo：会自动生成一张 300×300 的纯图形 SVG Logo，不用单独提需求；
- 字体：用系统字体，加载快；
- 没有 React / Vue / 构建工具——产出的就是可以**直接 FTP 上去就跑的静态网页**。

> 历史记录：每一次对话都会自动保存在左侧"历史会话"里，可以点回去重看完整对话与最终产物。

## 六、场景二：用现成模板批量上线一批网站

### 适用什么时候

- 一次要做 **十几到上百个** 域名落地页（典型场景：SEO 矩阵）；
- 不需要每个站都让 AI 重新设计，只需要**复用现有模板** + **改一下品牌字和 Logo**；
- 想要批量速度比 AI 全自动快**至少 10 倍**。

### 操作步骤

1. 「建站」Tab，模式胶囊切到 **「模板批量」**。
2. 输入框里**粘贴目标域名列表**——换行、空格、逗号、分号都能识别，例如：
   > foo.com
   > bar.com
   > baz.com
   > shop.example.io, store.example.dev
3. 右侧会出现"挑模板策略"，二选一：
   - **指定模板**：从下拉里选一个模板，所有域名都用它；
   - **自动挑模板**：轮询 / 随机。
4. 点发送，左侧对话窗会**一条条吐进度**：当前正在做哪个域名、用的哪个模板、产出在哪。
5. 跑完后到 **「网站库」** 即可看到新增的卡片。

预留截图位：

<p align="center">
  <img src="./images/screenshot-buildchat-batch.png" alt="模板批量模式：多行域名输入 + 进度流" width="800" />
</p>

### 它在做什么（粗略原理，给你心里有数）

对每一个域名：

1. 把选中的模板**整个目录复制一份**到新域名目录下；
2. **把所有出现源域名 / 品牌字的地方替换为新域名 / 新品牌字**（域名、大写、首字母大写、小写四种形式都会处理）；
3. **重新生成一张专属 Logo**，旧的几张 PNG Logo 删掉，节省后续上传带宽。

## 七、场景三：申请购买 / 解析域名（走飞书审批）

### 它怎么触发？

不需要切到别的页面——**直接在「建站」对话里用中文说一句话**就行，比如：

- "**帮我购买 foo.com、bar.com 这两个域名，1 年期，不需要 ICP 备案**"
- "**申请解析 api.foo.com 到 10.0.0.5**"

软件会自动识别"购买/采购/注册域名 + 域名"或"解析域名/域名解析"这些关键字，然后**在对话气泡里直接嵌入一张审批卡片**，并尝试从你的话里**抽取出域名 / 时长**自动预填。

### 卡片里你要做的事

预留截图位：

<p align="center">
  <img src="./images/screenshot-approval-card.png" alt="飞书审批卡片：发起人 + 域名表单 + 提交按钮" width="700" />
</p>

| 区块 | 说明 |
| :--- | :--- |
| 顶部 | 显示卡片类型（「域名购买」/「域名解析」）、飞书机器人头像；提交后会多一个**审批中**的小标签 |
| 发起人 | 已自动填好你的飞书姓名 + 用户 ID（不需要也无法修改） |
| 域名列表 | 表格形式，可以加行 / 减行 / 改内容（域名 + 时长/年）|
| 境内提供服务 / ICP 备案 | 二选一开关 |
| 申请说明 | 500 字以内的自由文本（可选） |
| 底部 | **取消卡片**（仅本地隐藏，不发送任何东西）+ **确认提交到飞书审批**（红色按钮） |

### 提交后会发生什么

1. 卡片右上角的标签从无 → **审批中**（蓝色）；
2. 你的飞书会收到一条「待我审批」（其实是审批流走到你或下一审批人）；
3. 应用在**后台每分钟自动查一次飞书**，看审批走到哪一步了：
   - 通过 → 标签变 **已通过**（绿色），同时**飞书会主动给你发一张结果通知卡片**；
   - 拒绝 → **已拒绝**（红色）；
   - 撤回 / 取消 → 对应灰色标签；
4. 在右上角头像菜单里有「**我的审批**」可以集中查看所有审批的当前状态。

> 撤销：在飞书后台撤销即可，应用里不放撤销按钮——避免误操作。

## 八、场景四：在网站库管理已生成的站点

顶部点 **「网站库」** Tab，可以看到所有已经生成的站点 + 系统自带的模板**平铺**成卡片。

预留截图位：

<p align="center">
  <img src="./images/screenshot-catalog.png" alt="网站库平铺：卡片 + 顶部筛选 + 批量操作栏" width="800" />
</p>

### 顶部筛选栏

| 控件 | 作用 |
| :--- | :--- |
| **网站库 (N)** | 当前可见卡片数量统计 |
| **搜索框** | 按域名 / 名称模糊搜索 |
| **全部 / 站点 (sites) / 模板 (templates)** | 一键切换只看哪一类 |
| **刷新按钮（↻）** | 重新从硬盘读取最新结果 |

### 单张卡片

- **正面是 Logo / 网站封面**；卡片底部有名称 + 类型标签（蓝 = site，紫 = template）。
- **悬停 / 选中** 时左上角会出现勾选框；
- **底部右侧三个图标按钮**（仅 site 卡片有）：
  - 👁 **预览**（在独立窗口打开当前网站本地版）；
  - ☁ **部署**（直接弹出部署对话框）；
  - ✏ **AI 修改**（回到对话窗，自动以"继续修改这个站"的模式开始）。
- **🗑 删除按钮**：**只在生成的站点（site）卡片上**显示；**模板（template）卡片永远不会显示删除按钮**，避免误删系统资产。

### 顶部批量操作（勾选 ≥ 1 张才会出现）

| 操作 | 说明 |
| :--- | :--- |
| **取消** | 清空所有选中 |
| **批量导出** | 把所有选中的站点打包导出到本地一个目录（弹出选择目录对话框） |
| **批量删除** | 把选中的所有 **站点** 永久删除；**勾到模板会自动被忽略**（不会删掉模板） |
| **批量部署** | 选中的所有 **站点** 串行上线到同一台服务器（templates 同样会被忽略） |

#### 关于批量删除（重要）

预留截图位：

<p align="center">
  <img src="./images/screenshot-catalog-batch-delete.png" alt="批量删除：红色按钮 + 二次确认弹窗" width="700" />
</p>

- 按钮显示的数字代表"**可删除的数量**"。例如你勾选了 5 张卡，其中 2 张是模板，按钮就会显示 **「批量删除（3）」**——只会删掉那 3 个站点。
- 点击后会弹出**二次确认弹窗**：
  - 列出即将删除的域名（前 6 个 + "等 N 个"）；
  - 明确写出 **"将永久删除……不可恢复，模板不会被删除"**；
  - 主按钮是红色的"删除（N）"。
- 确认后，应用会一个个删除并给出反馈：
  - 全部成功 → 顶部绿色"已删除 N 个网站"；
  - 部分失败 → 黄色提示并显示第一个失败原因；
- **删除是不可恢复的**——文件直接从你硬盘移除。如果有重要内容，先用"批量导出"备份。

> 单个删除：直接点卡片底部的 🗑 图标，弹窗逻辑与批量删除完全一致。

## 九、场景五：把网站一键部署到服务器

### 在哪里点

两个入口都能用：

- **网站库**里单张卡片的 ☁ 部署、或勾选多张点顶部的「批量部署」；
- 顶部 **「部署」** Tab 看历史日志、配置服务器。

### 部署对话框里的事

预留截图位：

<p align="center">
  <img src="./images/screenshot-deploy-dialog.png" alt="部署对话框：选择服务器 + 进度列表 + 实时日志" width="700" />
</p>

1. **目标服务器**下拉：列出当前已配置的所有服务器，**只有"已就绪"标签的才能选**（缺用户名或私钥的会被打成"不完整"，需要找 IT 同学补一下）。
2. 单域名 / 多域名都会列出来，点 **「建立 SSH 连接并开始批量上传」** 开始。
3. 进度区会**逐域名展示**：进度条、当前文件、字节数、成功 / 失败状态。
4. 跑完会**弹出气泡通知**：
   - 全部成功 → 绿色「部署完成：N/N 成功」；
   - 部分失败 → 黄色「部署完成：M/N 成功」；
   - 全部失败 → 红色提示。
5. 点 **「关闭」** 即可。

### 部署 Tab 能做什么

预留截图位：

<p align="center">
  <img src="./images/screenshot-deploy-page.png" alt="部署页：服务器卡片 + 部署日志列表" width="800" />
</p>

**上半页：服务器卡片**

- 上方展示**远端的两个固定路径**（网站根目录 + Nginx 配置目录），让你心里有数；
- 表格里能看到：IP / 端口 / 用户名 / 私钥状态 / 是否就绪；
- 操作：**编辑**（改 IP/端口/用户名）、**导入私钥**（系统弹窗选 .pem 文件）、**删除**；
- **就绪**：用户名 ∧ 私钥 都有 → 绿色就绪；缺则灰色"不完整"。

**下半页：部署日志卡片**

- 每一次部署都会落一条独立日志：哪个域名、哪台机器、几个文件、多少字节、用时；
- 点 **「查看」** 右边会**滑出一个抽屉**：
  - 上面展示这次部署的元信息；
  - 下面是等宽字体的**实时日志**，按颜色区分上传 / 删除 / 应用 / 完成 / 错误；
- 抽屉可以随时关闭，左侧表格会重新铺满整个屏幕。

### 部署可能出现的问题

- **认证失败**：通常是私钥或用户名不对——找 IT 同学；
- **502 / 404 访问不到**：应用只负责"上传 + 让 nginx 生效"，**不管 DNS 与防火墙**，需要检查域名解析是否指向了部署服务器、80 端口是否开放；
- **部署完成但是页面是空白的**：通常是 AI 生成的网站本身有问题——回到「网站库」点 ✏ 修改，让 AI 再来一轮。

## 十、右上角头像菜单 · 我的审批

点击右上角的飞书头像，能看到：

- **我的飞书账号信息**（姓名 + ID 后几位）；
- **我的审批列表**：把所有发起过的审批按状态分组（审批中 / 已通过 / 已拒绝 / 已撤回……）；
  - 鼠标移到某条上能看到**当时填的关键字段**；
  - 已完成的审批会在 90 天后自动停止后台轮询；
- **切换账号 / 退出登录**：用别的飞书号继续工作。

> 头像上的**红色数字角标** = 当前还没有走完审批流程的实例数；走完会自动清零。

## 十一、常见问题

<details>
<summary><b>① 启动后一直停在登录页 / 浏览器打不开授权页面</b></summary>

请：
1. 确认你电脑能正常打开浏览器；
2. 直接在浏览器地址栏访问 `https://open.feishu.cn`，能正常打开飞书首页即说明网络 OK；
3. 在登录页耐心等待"登录成功"提示，再回到应用；
4. 如果 5 分钟内没动作，应用会自动取消——重新点登录即可；
5. 多次失败请联系 IT 同学确认应用的飞书凭据是否过期。
</details>

<details>
<summary><b>② AI 生成的网站很慢 / 卡住了怎么办</b></summary>

- 顶部有 **「停止」** 按钮，点了之后会优雅地告诉云端停止；
- 已生成的部分文件**不会回滚**，可以在「网站库」点预览看看做到哪了；
- 想继续：在原对话里直接发新消息，会按"继续改这个站"的模式处理。
</details>

<details>
<summary><b>③ 模板批量做完后，网页里还显示老品牌字？</b></summary>

通常是因为模板里有些品牌字**和模板目录名不一致**（比如目录叫 `adliftlab.com` 但页面里写的是 `BoostBeacon`）。
请：
1. 找到出问题的模板对应的目录名；
2. 联系 IT 同学修一下，或者直接对该模板用 AI 修改的方式重做；
3. 不建议自己手工改模板。
</details>

<details>
<summary><b>④ 部署提示"认证失败 / Authentication failed"</b></summary>

不是你的操作问题，是服务器登录信息（用户名 / 私钥 / 口令）出了问题。请联系 IT 同学排查：
- 用户名（root / ubuntu / 等）是否正确；
- 私钥是否带口令；
- 私钥格式是否标准（不要用 PuTTY 自带的 .ppk）；
- 服务器侧的 `~/.ssh/authorized_keys` 是否还有这把公钥。
</details>

<details>
<summary><b>⑤ 部署成功了但是浏览器访问域名还是打不开 / 502 / 404</b></summary>

应用只完成"把文件上传到服务器 + 让 Nginx 生效"这一步，不管你的域名解析与防火墙。请检查：
1. 域名是否解析到了部署服务器的 IP（A 记录）；
2. 服务器 80 端口是否开放（云厂商安全组 / 防火墙）；
3. 找 IT 同学到服务器上看一下 `nginx -t` 是否通过。
</details>

<details>
<summary><b>⑥ 不小心删错了网站，能恢复吗？</b></summary>

**不能**。批量 / 单个删除都是直接从硬盘永久移除，没有回收站。请养成**删除前先批量导出做备份**的习惯。
</details>

<details>
<summary><b>⑦ 想批量删除一批站点，但其中混了几个模板，会不会一起被删？</b></summary>

不会。规则：
- **模板（template）的卡片根本不会显示删除按钮**；
- **批量删除按钮的数字**只会算"可删的站点数"——比如你勾了 5 张卡其中 2 张是模板，按钮显示「批量删除（3）」；
- 确认弹窗里也明确写了"模板不会被删除"。

放心删，模板是系统资产，UI 上做了多重保护，不会被误删。
</details>

<details>
<summary><b>⑧ 头像红色数字角标一直不消失</b></summary>

数字 = 还没走完审批流程的实例数。请检查：
1. 在飞书里那条审批是不是真的已经被批/拒了；
2. 等下一次后台自动查询（默认每 60 秒一次）就会自动消失；
3. 你也可以点击头像 → 我的审批，找到那条点"刷新"主动查询一次。
</details>
